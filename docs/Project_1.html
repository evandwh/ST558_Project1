Intro

This project creates a function for an API call to get US Census data from the correct URL. The function builds the URL, gets the data from the URL, then stores the data in a tibble.

We then create a way to summarize and plot our census data after creating the class census. The plotting is a boxplot.

Time Variables and Functions

The variables JWAP and JWDP are associated with times. Therefore, the first function we are going to need is a function that takes a string with a time interval in it and returns the midpoint of the time interval. I wanted to have all of the values eventually returned to as a time. It might make more sense have this returned as seconds since midnight, if we were trying to do some kind of summary on the variable. However, we have chosen not to summarize our time variables, and I enjoy seeing the actual time instead of need to do a computation to figure it out from a value of seconds.

interval_midpoint <- function(time_string) {
  
  if (is.na(time_string) || time_string == "0") {
    return(as_hms(NA))
  }

  pieces <- str_split(time_string, " to ", simplify = TRUE)

  start <- str_trim(pieces[1])
  end <- str_trim(pieces[2])
  
  start <- str_replace_all(start, " a.m.", " AM")
  start <- str_replace_all(start, " p.m.", " PM")
  
  end <- str_replace_all(end, " a.m.", " AM")
  end <- str_replace_all(end, " p.m.", " PM")
  
  start_time <- parse_date_time(start, orders = "I:M p")
  end_time <- parse_date_time(end, orders = "I:M p")

  midpoint <- start_time + (end_time - start_time) / 2
  as_hms(midpoint)
}

interval_midpoint("12:00 a.m. to 12:04 a.m.")
00:02:00

Our function seems to work as intended. It takes the string “12:00 a.m. to 12:04 a.m.”, determines the two times of 12:00 am and 12:04am, then returns the middle of the two times at 12:02am. It uses a 24 hour clock and reports it as 00:02.

State/Region Values

The next function we need is one that will take a given state or area and convert it to a number for the URL to know where to the look. There are three different possible geographic areas: STATE, REGION, DIVISION. Each has different codes. STATE uses two digits. I will use 37 for the state of North Carolina for demonstration purposes. REGION and DIVISION use single digit codes. There are 4 regions and 9 divisions. If someone uses an incorrect value for one of the three geographic areas, the function should stop and give the appropraite error message.

help_codes <- function(geo, geo_name) {
  state_fips <- c(
    AL = "01", AK = "02", AZ = "04", AR = "05", CA = "06",
    CO = "08", CT = "09", DE = "10", DC = "11", FL = "12",
    GA = "13", HI = "15", ID = "16", IL = "17", IN = "18",
    IA = "19", KS = "20", KY = "21", LA = "22", ME = "23",
    MD = "24", MA = "25", MI = "26", MN = "27", MS = "28",
    MO = "29", MT = "30", NE = "31", NV = "32", NH = "33",
    NJ = "34", NM = "35", NY = "36", NC = "37", ND = "38",
    OH = "39", OK = "40", OR = "41", PA = "42", RI = "44",
    SC = "45", SD = "46", TN = "47", TX = "48", UT = "49",
    VT = "50", VA = "51", WA = "53", WV = "54", WI = "55",
    WY = "56"
  )
  region_codes <- c(
    Northeast = "1",
    Midwest   = "2",
    South     = "3",
    West      = "4"
  )
  division_codes <- c(
    New_England        = "1",
    Middle_Atlantic    = "2",
    East_North_Central = "3",
    West_North_Central = "4",
    South_Atlantic     = "5",
    East_South_Central = "6",
    West_South_Central = "7",
    Mountain           = "8",
    Pacific            = "9"
  )
  if (geo == "STATE") {
    if ((geo_name %in% names(state_fips))) {
      code <- state_fips[[geo_name]]
    } else if (geo_name %in% state_fips) {
      code <- geo_name
    } else {
      stop("State code not found")
    }
  } else if (geo == "REGION") {
    if (geo_name %in% names(region_codes)) {
      code <- region_codes[[geo_name]]
    } else if (geo_name %in% region_codes) {
      code <- geo_name
    } else {
      stop("Region code not found")
    }
  } else if (geo == "DIVISION") {
    if (geo_name %in% names(division_codes)) {
      code <- division_codes[[geo_name]]
    } else if (geo_name %in% division_codes) {
      code <- geo_name
    } else {
      stop("Division code not found")
    }
  } else {
    stop("Invalid input. Incorrect geograph type.")
  }
  code
}

URL Builder

Now it is time to build our URL and GET() our results from said URL. Our function takes six different arguments.

  • Year, which is 2022 by default.

  • Num_Vars, our numeric varaibles, which is set to return AGEP and PWGTP by default. We want PWGTP to be returned every time. This will be taken care of within the function itself. We want at least one other numeric variable other than PWGTP. This is taken care of by checking the length of num_vars and comparing it to the number two (PWGTP plus one additional numeric variable.

  • Cat_Vars is the categorical variables, which is set to return SEX be default. We always want to have at least one Categorical variable in our function call. This is taken care of by checking the length of cat_vars and comparing it to the number 1.

  • Geo, which is STATE by default, is the type of geographical area we are interested in.

  • Geo_name, which is set to NC by default. The functions are set up to use either the state abbreviation or the corresponding code.

  • Key, which is set to my personal key default. Should my key fail to work at somepoint, or if someone wants to use their own key, they may adjust the key to their own personal key value.

single_year_rawdata <- function(year = 2022, 
                       num_vars = c("AGEP", "PWGTP"),  #AGEP and PWGTP by default
                       cat_vars = c("SEX"), #SEX by default
                       geo = "STATE",       #State by default
                       geo_name = "NC",
                       key = "70c77fa9cc244f5a9fb3c46ed731835d132c0670") {
  num_vars <- toupper(num_vars)
  cat_vars <- toupper(cat_vars)
  geo <- toupper(geo)
  if (year > 2022 || year < 2010) {
    stop("Year must be between 2010 and 2022, inclusive.")
  }
  if (!("PWGTP" %in% num_vars)) {
    num_vars <- c(num_vars, "PWGTP")  #Always want PWGTP
  }
  if (length(num_vars) <= 1) {
    stop("Please provide more than one numerical variable.")
  }
  for (k in num_vars) {
    if (!(k %in% c("AGEP", "GASP", "GRPIP", "JWAP", "JWDP", "JWMNP", "PWGTP"))) {
      stop("Please choose only valid options for num_vars.")
    }
  }
  for (k in cat_vars) {
    if (!(k %in% c("FER", "HHL", "SCH", "SCHL", "SEX"))) {
      stop("Please choose only valid options for cat_vars.")
    }
  }
  if (length(cat_vars) == 0) {  #Must have at least one categorical variable.
    stop("Must have at least one categorical variable.")
  }
  if (!(is.null(geo))) {
    if (!(toupper(geo) %in% c("REGION", "DIVISION", "STATE"))) {
      stop("Incorrect Geography provided")
    }
    geo_string <- paste0("&for=",geo)
  } else {
    geo_string <- ""
  }
  if ((geo_string == "") && (!(is.null(geo_name)))) {
    stop("Cannot give a location without a specificed geography type")
  }
  if (geo_name == "ALL") {
    code <- "*"
  }
  else {
    code <- help_codes(geo, geo_name)
  }
  geo_string <- paste0(geo_string,":",code)
  num_string <- paste(num_vars, collapse = ",")
  cat_string <- paste(cat_vars, collapse = ",")
  url <- paste0("https://api.census.gov/data/",
                year,
                "/acs/acs1/pums?get=",
                cat_string,
                ",", #cat_var and num_var always have at least one element
                num_string,
                geo_string,
                "&key=",
                key)
  print(url)
  results <- GET(url)
}

Summarizing

Now that we have our data, let’s summarize it!

Weighted Helper Functions

PWGTP is considered a weight of many different responses. Therefore, tradition mean and standard deviation functions will not work. We need to create two functions for the the weighted mean and weighted standard deviation.

weighted_mean <- function(col, weight) {
  sum(col * weight, na.rm = TRUE) / sum(weight, na.rm = TRUE)
}

weighted_sd <- function(col, weight) {
  mu <- weighted_mean(col, weight)
  sqrt(
    sum(col^2 * weight, na.rm = TRUE) / sum(weight, na.rm = TRUE) - mu^2
  )
}

Summary Function

We want to return the weighted mean and standard deviation of all numeric data and the counts of the categorical data.

summary.census <- function(data, num_vars = NULL, cat_vars = NULL) {
  
  if (is.null(num_vars)) {
    num_vars <- data |> 
      select(where(is.numeric)) |> 
      names()
    num_vars <- setdiff(num_vars, c("year","PWGTP"))
  }
  
  if (is.null(cat_vars)) {
    cat_vars <- data |>
      select(where(is.factor)) |>
      names()
  }
  
  numerical_summary <- data |> 
    summarise(
      across(
        all_of(num_vars), 
        list(
          mean = ~ weighted_mean(.x, PWGTP),
          sd   = ~ weighted_sd(.x, PWGTP)
          ),
        .names = "{.col}_{.fn}")
    ) |>
    pivot_longer(
      everything(),
      names_to = c("variable", "statistic"),
      names_sep = "_",
      values_to = "value"
    )
  
  categorical_summary <- data |>
    pivot_longer(
      cols = all_of(cat_vars),
      names_to = "variable",
      values_to = "level"
    ) |>
    count(variable, level)
  
  full_summary <- list(
    "Numeric_Summary" = numerical_summary, 
    "Categorical_Summary" = categorical_summary)
  full_summary
}

Plotting Our Data

Finally, we want to be able to plot our data in a good format. This function takes one numeric and one categorical variable to and weighted by PWGTP. The x-axis will be split by the different factors of the categorical variable. A boxplot will be created for each factor of the categorical data for the given numeric variable. For better readability, the x-axis labels have been rotated 45 degrees.

plot.census <- function(data, num_var, cat_var) {
  ggplot(
    data,
    aes(
      x = get(cat_var),
      y = get(num_var),
      weight = PWGTP
    )
  ) +
    geom_boxplot() +
    labs(
      x = cat_var,
      y = num_var,
      title = paste(num_var, "by", cat_var)
  ) + 
    theme(
      axis.text.x = element_text(angle = 45, hjust = 1)
  )
}
